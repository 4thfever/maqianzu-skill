# 上游语料目录观察

本文件记录 Phase 3 对 `bedtimenews-archive-contents` 目录形态的实际观察，用于指导后续 chunk 切分和索引生成。

## 当前观察结果

### `main`

`main` 目录不是按年份组织，而是按期次区间分桶，例如：

- `main/1-100/1.md`
- `main/1-100/10.md`
- `main/1001-1100/...`

这意味着后续处理 `main` 时：

- 不能直接依赖年份目录
- 更适合把“期次号”和文件内元数据当作主键

### `livestream`

`livestream` 目录组织方式更复杂，当前至少包含两层：

- 顶层年份索引文件，例如 `livestream/2022.md`
- 年份目录与月份/日期分层文件，例如 `livestream/2022/11/15-1.md`

这意味着后续处理 `livestream` 时：

- 需要区分“导航索引页”和“实际内容页”
- 不应把 `2022.md`、`2023.md` 这类年度索引页直接当作普通节目正文

## 对后续 Phase 4 的直接影响

- `main` 和 `livestream` 不能用同一套“目录即元数据”的简化假设处理
- chunk 切分前需要先识别内容页与索引页
- `livestream` 可能需要先过滤汇总页，再进入正文切分
- `main` 更适合基于文件 front matter 或正文标题抽取元数据

## 当前建议

进入 Phase 4 前，建议先补一层预处理规则：

1. 识别索引页、导航页、正文页
2. 为 `main` 和 `livestream` 分别设计元数据抽取策略
3. 再进入统一的 chunk 切分流程
